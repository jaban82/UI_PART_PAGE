We necessary technique

우리 파트에서는 아래 기술들을 이용한 마크업 업무를 진행하고 있으며,
아래 기술에 대한 연구개발을 지속적으로 진행하고 있습니다.

  • HTML

    HTML(Hypertext Markup Language)는 웹문서를 만들기 위해 사용하는 기본적인 프로그래밍 언어로 업무진행에 있어서 가장 기본적으로 요구되는 기술 입니다.

  • CSS

    CSS(cascading style sheets)는 웹문서의 전반적인 스타일을 저장하는 스타일 시트로서 마크업 업무간 정확한 스펙 및 구현방법의 습득이 필요한 기술 입니다.

  • Photoshop

    마크업 업무는 동일 업무라도 그 구현방법이 모두 다릅니다. 따라서 전달된 디자인 가이드에 따른 포토샵 기술은 마크업 업무자에게도 필요한 기술 입니다.

  • web standards

    W3C(World Wide Web Consortium)에서 권고한 표준안을 이용하여 서로다른 환경에서도 동일한 결과물을 볼 수 있도록 하는 기술입니다.

FAQ

우리 파트에 대해서 자주 묻는 질문들을 모았습니다.

'웹표준'이란 무엇인가요?

웹 표준이란

월드 와이드 웹의 측면을 서술하고 정의하는 공식 표준이나 다른 기술 규격을 가리키는 일반적인 용어입니다. 웹사이트나 웹페이지가 웹 표준을 준수한다는 것은 일반적으로 올바른 HTML,CSS, 자바스크립트를 사이트나 페이지가 가지고 있다는 것을 뜻합니다.

'웹접근성'이란 무엇인가요?

웹접근성이란

사용자의 신체적 특성이나, 성별, 나이, 지식 수준, 체험과 같은 제한 사항을 고려하여 가능한 많은 사용자가 불편 없이 이용할 수 있도록 서비스를 만들어 제공하고 이를 평가 할 때 쓰이는 말입니다. 접근성이 높다는 것은 이러한 제한 사항을 가진 사용자도 불편 없이 사용할 수 있다는 것을 뜻하며 접근성이 낮다는 것은 어떠한 제한 때문에 사용하기 불편하거나 사용할 수 없을 때를 말합니다.

업무를 진행하기위해 필요한 스킬은 HTML/CSS 외 무엇이 있나요?

프론트 개발자로서 역량을 키우기 위해 javascript, jquery 등의 기술에 대해 파트내에서 학습하고 있습니다.(일부 업무는 javascript, jquery 이미 활용중)

JavaScript/jquery는 웹페이지를 동적으로 제어하기 위해서 고안된 언어로 정적인 마크업 산출물을 동적으로 움직일 수 있도록 제어합니다.

업무는 어떻게 진행이 되나요?

웹개발의 큰 프로세스는 '기획->디자인->마크업->개발->QA->오픈'으로 진행됩니다.

여기서 우리파트는 '마크업' 영역을 담당하고 있습니다. 단순 '마크업'영역뿐 아니라 이후 프로세스(개발-QA-오픈)에서 마크업 기술지원을 담당하고 있습니다.

마크업에 대한 학습을 진행하고 싶습니다. 참고할 사이트가 있을까요?
